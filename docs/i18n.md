# Connpass — 다국어(i18n) 가이드

프론트엔드는 KO / EN / ZH 3개 언어를 지원한다.
번역 문자열은 `frontend/locales/` 아래 언어별 JSON 파일로 관리되고,
`frontend/js/i18n.js`가 런타임에 비동기 로드한다.

---

## 파일 구조

```
frontend/
├── js/
│   └── i18n.js              ← 언어 로드 및 전역 API
└── locales/
    ├── ko.json              ← 한국어 (기본)
    ├── en.json              ← 영어
    └── zh.json              ← 중국어 (간체)
```

---

## 전역 API

`i18n.js`는 `window`에 4개의 함수/프라미스를 노출한다.

### `t(key, vars?)`

현재 언어의 번역 문자열을 반환한다.

```js
t('nav.newChat')                        // "＋ New Chat"
t('llm.modelsFound', { n: 3 })          // "3 models found"
t('model.offline', { model: 'GLM4.7' }) // "GLM4.7 model is currently offline."
```

- `{placeholder}` 형식으로 변수 치환.
- 현재 언어에 키가 없으면 `ko` 폴백 → 그래도 없으면 키 자체를 반환 (UI 깨짐 없음).

### `setLang(lang)`

언어를 전환한다.

```js
setLang('en')   // 'ko' | 'en' | 'zh' 중 하나
```

내부 동작:
1. `localStorage`의 `ui-lang` 값 갱신
2. `<html lang>` 속성 갱신 (`zh` → `zh-CN`)
3. `.lang-btn` 버튼의 `active` 클래스 갱신
4. `applyI18n()` 호출 → 모든 `data-i18n` 요소 일괄 갱신
5. `app.js`의 `applyLang()` 호출 → 동적 UI 갱신 (placeholder 등)

### `applyI18n()`

`data-i18n*` 속성이 있는 모든 DOM 요소를 현재 언어로 갱신한다.
언어 전환 시 자동 호출되며, 직접 호출할 일은 거의 없다.

### `i18nReady`

모든 locale JSON 로드 완료를 알리는 `Promise`.

```js
await window.i18nReady;
// 이제 t() 사용 가능
```

`app.js`의 `init()` 함수가 이 프라미스를 `await`한 후 언어를 적용하므로
**페이지 로드 시 번역이 누락되는 현상이 발생하지 않는다.**

### `getCurrentLang()`

현재 활성 언어 코드를 반환한다.

```js
getCurrentLang()  // 'ko' | 'en' | 'zh'
```

## HTML에서 번역 적용하기

정적 HTML 요소에는 `data-i18n` 속성을 추가한다.
`applyI18n()` 호출 시 해당 키의 번역값이 `innerHTML`로 삽입된다.

```html
<!-- 텍스트 내용 -->
<button data-i18n="nav.newChat"></button>

<!-- placeholder -->
<input data-i18n-placeholder="input.placeholder" />

<!-- title (tooltip) -->
<button data-i18n-title="input.sendTitle"></button>
```

> `data-i18n` 값은 `innerHTML`로 적용되므로 `<br>`, `<code>`, `<span>` 등 인라인 태그 사용 가능.

---

## JS에서 번역 적용하기

JS 코드에서는 `window.t()` 또는 `t()`를 직접 호출한다.

```js
// 단순 문자열
showToast(t('settings.saved'));

// 변수 포함
showToast(t('llm.modelsFound', { n: models.length }));

// confirm 다이얼로그
if (!confirm(t('session.deleteConfirm'))) return;

// 동적 DOM 생성
el.textContent = t('memory.loading');
```

**chat.js처럼 i18n 로드 타이밍이 불확실한 경우** 폴백 패턴을 사용한다:

```js
const label = window.t ? window.t('thinking.label') : '생각 중...';
```

## 번역 키 추가하기

1. **세 locale 파일 모두에 동일한 키를 추가한다.**

   `frontend/locales/ko.json`
   ```json
   "example.newKey": "새로운 텍스트"
   ```

   `frontend/locales/en.json`
   ```json
   "example.newKey": "New text"
   ```

   `frontend/locales/zh.json`
   ```json
   "example.newKey": "新文本"
   ```

2. **코드에서 사용한다.**

   ```js
   t('example.newKey')
   ```

> 키는 `category.camelCase` 형식을 권장한다. (예: `llm.fetchModels`, `memory.colDate`)

---

## 새 언어 추가하기

1. `frontend/locales/<lang>.json` 파일을 생성하고 모든 키에 번역을 추가한다.
   - 시작점: `ko.json`을 복사해서 번역 교체.
   - `time.locale` 키를 해당 언어의 BCP 47 태그로 설정 (예: `"ja-JP"`).

2. `frontend/js/i18n.js`의 `SUPPORTED` 배열에 언어 코드를 추가한다.

   ```js
   const SUPPORTED = ['ko', 'en', 'zh', 'ja'];  // 추가
   ```

3. `frontend/index.html`에 언어 버튼을 추가한다.

   ```html
   <button class="lang-btn" data-lang="ja">JA</button>
   ```

4. `setLang()` 내부의 `html lang` 처리를 필요에 따라 수정한다 (현재 `zh` → `zh-CN` 매핑 존재).

| 카테고리 | 키 접두사 | 설명 |
|----------|-----------|------|

| WebSocket | `ws.*` | 연결 상태 메시지 |
| 번역 모드 | `translate.*` | 번역 배지, 버튼 |
| 빈 화면 | `empty.*`, `qp*.* ` | 초기 화면 문구, 빠른 프롬프트 |
| 입력창 | `input.*` | placeholder, 버튼 title, 힌트 |
| RAG | `rag.*`, `chip.*` | RAG 레이블 |
| 설정 공통 | `settings.*` | 저장/저장됨 |
| LLM 설정 | `llm.*` | 모델 등록 폼 |
| MCP 설정 | `mcp.*` | MCP 서버 관리 |
| Jira 설정 | `jira.*` | Jira 서버 관리 |
| Gerrit 설정 | `gerrit.*` | Gerrit 서버 관리 |
| 스킬 | `skill.*` | 스킬 관리 |
| 번역 설정 | `translateSettings.*` | 번역 모드 설정 |
| Agent.md | `agentmd.*` | 개인화 설정 |
| UI 설정 | `ui.*` | 테마, 폰트 등 |
| 메모리 | `memory.*` | 장기 기억 관리 |
| 채팅 렌더러 | `thinking.*`, `compaction.*`, `error.*` | 채팅 블록 레이블 |
| 시간 표현 | `time.*` | 상대 시간, locale 태그 |

전체 키 목록은 `frontend/locales/ko.json`을 참고한다.

---

## 주의 사항

- `ko.json`이 기본(기준) 파일이다. 새 키 추가 시 `ko.json`을 먼저 작성하고 EN/ZH를 동기화한다.
- locale JSON은 Express의 정적 파일 미들웨어로 서빙된다 (`/locales/` → `frontend/locales/`).
- `window.t()`는 i18nReady 이전에도 호출할 수 있지만, locale 데이터 로드 전에는 `ko` 폴백조차 없어 키 자체가 반환된다. 초기화 흐름상 문제가 생기면 `await window.i18nReady`로 보호한다.
