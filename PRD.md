# PRD — IntelliSearch
> Product Requirements Document
> 대상: BT/WiFi 엔지니어링팀 | 작성: 2026-02-28

---

## 1. 배경 및 목적

### 문제
- BT/WiFi 스펙 문서, Jira 이슈, Gerrit 코드리뷰, Confluence 위키가 파편화되어 정보 탐색에 시간이 많이 소요
- 유사 이슈 발생 시 과거 RCA(Root Cause Analysis) 재탐색이 어려움
- 반복적인 주간 이슈 요약, 코드리뷰 초안 작성 등 단순 업무에 시간 낭비

### 목표
엔지니어가 자연어로 문서를 검색하고, 업무 도구를 조작하고, 반복 작업을 자동화할 수 있는 **사내 전용 AI 어시스턴트** 구축.

---

## 2. 사용자 및 사용 시나리오

### 대상 사용자
- BT 펌웨어 엔지니어 (주 사용자)
- WiFi 엔지니어
- 팀 리더 (주간 리포트 자동화)

### 핵심 시나리오

**시나리오 A — 이슈 분석**
```
"BT-4821 분석해줘"
→ Jira에서 이슈 조회
→ 관련 Gerrit 커밋 검색
→ BT Spec 관련 섹션 검색
→ 원인·영향·해결 방향 요약 응답
```

**시나리오 B — 유사 이슈 검색 (RCA)**
```
로그 파일 첨부 + "이거랑 비슷한 이슈 있었어?"
→ HCI 에러 패턴 추출
→ RCA Map에서 유사 이슈 검색
→ 과거 해결 방법과 함께 연결된 이슈 목록 응답
```

**시나리오 C — 번역**
```
영어 스펙 텍스트 붙여넣기 + 번역 모드 ON
→ tool 호출 없이 LLM 직통 한국어 번역
→ 기술 용어 원문 유지
```

**시나리오 D — Gerrit 코드리뷰 도우미**
```
"CR-1234 리뷰해줘"
→ Gerrit에서 diff 조회
→ BT Spec 기준으로 리뷰 코멘트 초안 작성
```

**시나리오 E — 주간 이슈 요약 (Cron)**
```
매주 월요일 09:00 자동 실행
→ BT 프로젝트 미해결 이슈 조회
→ 담당자별 정리 + 지연 이슈 하이라이트
→ 요약 리포트를 세션에 전달
```

## 4. 비기능 요구사항

- **응답 지연**: 스트리밍 첫 토큰 < 2초 (일반 모드), < 1초 (번역 모드)
- **동시 사용자**: 팀 내 ~20명 동시 세션 지원
- **보안**: 사내망 전용, 외부 LLM API 미사용
- **데이터**: 세션/대화 이력 SQLite 영구 저장
- **브라우저**: Chrome 최신 버전 (데스크탑 전용)
